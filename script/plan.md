# 구현 계획: Competitive Taiko Rating 스크립트

본 문서는 `script/index.ts`의 `main` 함수 구현을 위한 상세 계획을 담고 있습니다.

## 1. 사전 준비 (Setup)
- [x] `glicko2-lite` 라이브러리 설치 (`bun add glicko2-lite`)
- [x] `script/module/` 디렉토리 생성 및 기존 모듈 복구/고도화
    - [x] `db.ts`: 데이터베이스 스키마 정의 및 쿼리 함수 (QueryBuilder 활용)
    - [x] `hirobaCompe.ts`: 히로바 대회 생성 및 랭킹 크롤링 로직
    - [x] `getAccounts.ts`: `accounts.json`에서 계정 정보 로드
- [x] 곡 목록 데이터 (`songs.json`) 정의
    - [x] 그룹 A, B, C로 나누어진 곡 리스트 필요

## 2. 메인 로직 상세 (main 함수)

### 2.1. 데이터 초기화 (Step 1, 2)
- [x] `getLatestCompetition()`: 가장 최근 진행된 대회(session) 정보를 가져옴.
- [x] `getCurrentSeason()`: 현재 활성화된 시즌 정보를 가져옴.

### 2.2. 시즌 관리 (Step 3)
- [ ] **시즌 갱신 조건**: 오늘이 해당 월의 1일이거나, 이전 대회의 시즌이 종료되었을 경우.
- [ ] **신규 시즌 생성**:
    - 시즌 번호 증가.
    - `startDate`: 해당 월의 1일.
    - `endDate`: 해당 월의 마지막 날.
    - `season` 테이블에 삽입.

### 2.3. 실행 조건 확인 (Step 4)
- [ ] `checkDate` 확인: 최근 대회의 `checkDate`가 오늘과 같거나 이전인지 확인.
    - `luxon`의 `DateTime`을 사용하여 비교.
    - 조건 미충족 시 스크립트 종료.

### 2.4. 결과 수집 및 집계 (Step 5, 6)
- [ ] **히로바 대회 아이디 조회**: `hiroba_competition` 테이블에서 현재 시즌/세션에 해당하는 모든 `compeId` 조회.
- [ ] **랭킹 크롤링**:
    - 각 `compeId`에 대해 `hiroba-js`를 사용하여 랭킹 페이지 크롤링.
    - 각 플레이어의 태이코 번호(`taikoNo`)와 점수(`score`) 추출.
- [ ] **점수 통합**:
    - 동일한 `taikoNo`를 가진 플레이어들의 점수를 합산 (Map 사용).
- [ ] **결과 저장**: `hiroba_competition_history` 테이블에 플레이어별 최종 점수 기록.

### 2.5. 플레이어 그룹화 (Step 7)
- [ ] **대상 선정**: 이번 세션에 참여한 모든 플레이어의 현재 레이팅 정보(`rating`, `RD`, `Vol`) 조회.
- [ ] **정렬**: 레이팅(`rating`) 내림차순으로 정렬.
- [ ] **그룹 분할 알고리즘**:
    1. 기본 그룹 크기: 4~6명 지향 (최소 2명, 최대 8명).
    2. 정렬된 리스트를 순회하며 그룹 생성.
    3. 마지막에 남은 인원이 2명 미만일 경우, 바로 앞 그룹에 병합 (이 경우 최대 8명을 초과할 수 있음).
    4. README 명세에 따라 레이팅 차이가 너무 큰 경우 분리 로직 고려.

### 2.6. 레이팅 계산 (Step 8)
- [ ] **Glicko-2 업데이트**:
    - 각 그룹 내에서 1:1 매치 시뮬레이션.
    - `점수_A > 점수_B` 이면 A 승리, B 패배.
    - `glicko2-lite` 라이브러리를 사용하여 새로운 `rating`, `RD`, `Vol` 산출.
- [ ] **결과 반영**:
    - `rating` 테이블 업데이트 (현재 레이팅).
    - `rating_history` 테이블에 세션별 기록 삽입.

### 2.7. 차기 대회 생성 (Step 9)
- [ ] **세션 정보 갱신**: `session` 번호 1 증가.
- [ ] **곡 선정**:
    - 곡 목록 데이터에서 그룹 A, B, C 별로 랜덤하게 1곡씩 선정 (총 3곡).
- [ ] **히로바 대회 개설**:
    - `accounts.json`에 등록된 계정들을 순회하며 `openHirobaCompe` 호출.
    - 생성된 `compeId`들을 `hiroba_competition` 테이블에 저장.
- [ ] **대회 정보 저장**: `competition` 테이블에 신규 레코드 삽입.
    - `startDate`: 오늘.
    - `endDate`: 오늘 + 1일.
    - `checkDate`: 오늘 + 2일.
