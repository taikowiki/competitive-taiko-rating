# Competitive Taiko Rating

太鼓の達人のオンライン大会の成績を基に、実力を算出する **C-Rating (Competitive Rating)** システムです。

---

## 📋 プロジェクト概要
従来の成果ベースのレーティングシステムとは異なり、**オンライン大会の成績に特化したレーティングモデル**を目指しています。

*   **[P-Rating (Performance Rating)](https://rating.taiko.wiki)**: 既存のプレイデータを基に算出される実力指標。
*   **C-Rating (Competitive Rating)**: [ドンだーひろば](https://donderhiroba.jp)のオンライン大会の成績を基に算出される実力指標。大会の成績が高いほど、より高いレーティングが算出されます。

## 🏆 運用方式: オンライン大会
公平な実力測定を行うため、短い周期でオンライン大会を運営します。

*   **大会の進行**: 複数のオンライン大会を開催し、成績を統合的に収集します。
*   **課題曲の選定**: 毎周期リセットされる3つのグループ（A、B、C）からランダムに楽曲を選出し、公平性を維持します。

## 📈 レーティング算出ロジック
C-Ratingは、プレイヤーのグループ化とGlicko（グリコ）アルゴリズムを用いて計算されます。

### 1. プレイヤーのグループ化 (Grouping)
レーティングの識別性を確保するため、以下のルールに従ってプレイヤーをグループ化します。
*   **基準**: 既存のレーティングでソートした後にグループ化。
*   **バランスの維持**: グループ内の最高・最低レーティングの差が、特定の閾値（しきいち）以内に収まるように管理。
*   **制限事項**: 1グループあたり最小2名、最大8名に制限。
*   **例外処理**: どのグループにも属せなかったプレイヤーは、レーティングの差が最も小さいグループへ統合。（この場合のみ、8人を超えてもよいものとする。）

### 2. レーティングの更新 (Glicko Algorithm)
グループ化されたデータを基に、**Glickoアルゴリズム**を用いてレーティングを更新します。
*   **相対評価**: グループ内のすべてのプレイヤーとの試合結果をシミュレーションして更新。
*   **勝敗の判定**: 自分より順位が高いプレイヤーには「敗北」、低いプレイヤーには「勝利」したものとみなして、相対的な実力を算出。
